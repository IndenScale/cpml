# CPML — Construction Process Modeling Language

施工过程建模语言。声明式 DSL，用于细粒度、全维度地描述施工过程，由编译器执行静态分析、求解可行区间、暴露作业冲突与资源竞争。

## 定位

开源确定性验证编译器 —— 用于施工过程的静态分析。

## 核心特点

1. **建模对象：施工过程** — 不是产品结构（BIM），不是几何（3D），而是建造行为本身
2. **细粒度、全维度** — 作业、资源、空间、时间、逻辑关系，全要素覆盖
3. **声明式** — 描述"是什么"和"约束条件"，而非"怎么做"；编译器负责求解可行区间与冲突


## 基础概念

### 原子活动（Atomic Activity）

施工过程的基本建模单元。
具有 timespan，硬碰撞范围，软碰撞范围。
是探针和投影的容器。

### 探针（Probe）

建模活动的前置条件（precondition）。
包含采样范围和采样逻辑两个要素。
本质上是相对活动坐标的物理范围，对关注场景中特定维度的场进行采样，然后进行断言。

### 投影（Projection）

建模活动的贡献/扰动（contribution/disturbance）。

**几何表示：** 使用相对原子活动的几何体（AABB、球、锥等）或其布尔组合（union/intersect/subtract）定义影响范围。

**遮挡处理：** 投影受场景中阻挡体（楼板、墙体、柱等）影响，默认使用 raycasting 进行遮挡判别，被遮挡区域从影响范围中剔除。

### 场（Field）

时空局域的概念，用于介导活动之间的相互作用。
场的叠加规则可以定义，包含线性叠加、线性扣减（资源容量场）、min、max等算法。

### 系列活动（Activity Series）

由于活动的前置条件和影响会随时间发生变化，因此使用系列活动建模活动的时变特性。系列活动由一组离散原子活动组成，避免连续时间仿真。通过 Serial ID 进行关联。

### 场景（Scene）

特定时间点的时空切片，容纳原子活动和场，产生诊断。

### 诊断（Diagnostic）

每个关键帧中，探针断言失败时产生的告警。核心信息：

- **定位：** 什么时间、什么空间区域、哪个原子活动的哪个探针
- **根因：** 某维度场强超出阈值（过高/过低）
- **归因（Blame）：** 该位置场强的贡献者列表，追溯到具体的原子活动和投影/结构

**分级：** `debug` → `info` → `warning` → `error` → `fatal`，按严重度递进。

### 指标（Metric）

施工方案的代价或分数的积分。每个关键帧的诊断信息不仅产生告警，还按规则累加到指标上（如增加成本、增加风险指数），最终输出一个随时间累积的量化评价函数。

### 结构（Structure）

语法糖。结构是原子活动的一个属性，同时产生探针和投影——即同时对场进行采样（作为探针）和向场注入（作为投影）。本质是将常见的、对称的场-探针模式进行简化。

- **生命周期：** 结构与所属原子活动绑定。原子活动开始时结构生效，原子活动结束时结构移除。永久结构属于一个生命周期贯穿整个施工期的原子活动；临时结构（如脚手架、模板）属于对应施工活动。
- **碰撞敏感：** 结构对软碰撞和硬碰撞均敏感，编译器自动生成两类探针和投影。

### 概念关系

- 原子活动位于场景中
- 探针采样场以判定前置条件，断言失败时产生诊断
- 投影将活动的扰动注入场，是场强的贡献者
- 场景 = 一组原子活动 + 当前场状态，在特定时间切片上求值
- 系列活动将原子的时变行为离散化，每个时间步对应一个原子活动
- 结构是原子活动的语法糖属性，同时产生对称的探针和投影
- 诊断包含分级 + 归因（Blame），指向具体贡献者
- 指标按关键帧累积诊断影响，形成代价/风险的时间积分
