# ADR: 场求值引擎

## 背景

四种场类型（Capacity、Occupancy、Scalar、Presence）具有不同的求值语义——部分非持久（每帧重新计算），部分需要跨关键帧携带状态。

## 决策

### 持久性分类

**非持久场（Capacity、Occupancy）：** 每帧仅根据当前活跃投影重新计算。活动结束后其贡献立即消失。

**持久场（Scalar、Presence）：** 状态跨关键帧持续保留。活动的投影贡献在活动结束后依然存在，直至被后续投影显式覆盖、替换或过期。

### 统一的按区域存储

所有场类型均按空间区域（`region_key`）组织状态，探针求值时按区域重叠过滤。

```
非持久场：每帧从活跃投影重建 region → value 映射
持久场：  在 PersistentState 中维护 region → value 映射，跨帧携带
```

**PersistentState 结构：**

```
scalar:   field_name → (region_key → value)
presence: field_name → (region_key → (record_key → record))
```

**探针求值流程（对所有场类型统一）：**

1. 取得探针的 `region_key`（世界空间 AABB 的字符串表示）
2. 找到与该 region 空间重叠的所有状态条目（精确匹配 + AABB 重叠检测）
3. 用场类型的叠加算子合并重叠区域的值
4. 叠加当前活跃投影中几何区域重叠的贡献
5. 应用断言

### Scalar 场：合并重叠持久区域

当多个持久状态区域的 AABB 与探针区域重叠时，使用场的算子合并：
- `max`：取所有重叠区域的最大值
- `min`：取最小值
- `sum`：求和
- `replace`：第一个找到的值（持久状态条目无序）

### Presence 场：按区域存储记录

每条记录属于注入投影的 `region_key`。探针查询时收集所有空间重叠区域的记录，在其中查找匹配项。

### 容量场

对所有活跃投影值求代数和。正值 = 供给，负值 = 消耗。默认值：0。仅计算几何区域与探针区域重叠的投影。

### 占用场

对所有活跃投影贡献做逻辑或。按 `kind` 区分两个独立实例（hard 和 soft）。默认值：未占用。
排除自身活动的投影（自排除规则）。

## 后果

- 不同空间区域的持久状态正确隔离，避免假阳性
- `foundation` 区域强度与 `column_base` 探针通过空间重叠自然关联
- region_key 基于浮点坐标，极微小的几何差异可能导致不匹配——使用 1 位小数精度做 hash
- 重叠检测 O(n)，n 为某场的持久状态条目数；v1 场景规模下可接受
