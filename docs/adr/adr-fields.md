# ADR: 场类型定义与计算规则

## 背景

CPML 通过场（Field）介导活动之间的相互作用。投影（Projection）向场注入贡献；探针（Probe）对场进行采样并断言条件。本文定义场的封闭类型集合、值域、叠加算子及探针断言规则。

设计要求：

- **足够封闭**，使编译器明确知道自己在算什么
- **足够开放**，允许随需求增长引入新场类型和新算子

## 场模型

每个场实例具有：

- **类型**（以下定义的四种之一）
- **空间范围**——场存在的区域（全局、某个施工区段、特定体积）
- **叠加算子**——多个投影的贡献如何合并
- **当前值**——在给定关键帧上，对所有活跃投影施加叠加算子的结果

投影注入值；探针在采样范围内对合成值进行采样并断言。探针不做任何修改——资源消耗通过负投影建模（见容量场）。

### 统一接口原则

所有外部信息进入 CPML 的唯一入口是**活动**，而非绕过活动体系直接向场写入记录。

这意味着：供应链信息（物料到场）建模为物流到货活动；行政审批信息建模为审批活动；设备/班组资源配置建模为资源调度活动。每个活动携带投影，将信息注入对应场。

这样做的好处：数字孪生、供应链系统、OA 审批系统、资源管理系统等外部数据源，只需声明各自领域的活动及其投影，即可统一接入 CPML 的冲突检测管线——无需为每种外部信息设计单独的数据通道。

## 场类型

### 1. 容量场（CapacityField）

**用途：** 建模有限的、可消耗的资源——电力、供水、通道通行量、劳动力、材料库存、预算。

**值域：** ℝ（实数）。正值 = 供给，负值 = 消耗。

**叠加算子：** 代数和。

```text
V(p) = Σ projection_i(p)   对该关键帧所有活跃 i
```

任一点处的场值为覆盖该点的所有投影的代数和。无投影覆盖的点 → 值为 0。

**空间行为：** 每个投影声明自身的空间范围。供给投影可以是全局的（如全场供电）或局部的（如某区域发电机）。消耗投影通常是局部的（活动作业面）。

**探针断言：** `V(采样范围) ≥ 阈值`

- 探针在其声明区域内采样场值
- 计算值为该区域内所有投影贡献的代数和
- 低于阈值 → 断言失败

**诊断等级：** 默认 `error`，可覆盖。

**归因：** 采样区域内所有贡献者，按幅度排序，包含正贡献（供给不足）和负贡献（消耗过大）。

**示例：**

- 供电投影注入 +500kW；三个活动投影消耗 -150kW, -200kW, -180kW → 剩余 = -30kW → 断言失败
- 通道通行量：通道容量 N，每个通过活动消耗 1 单位

### 2. 占用场（OccupancyField）

**用途：** 建模几何占用——物理碰撞（硬碰撞）和风险/安全区域（软碰撞）。

**值域：** 空间点上的布尔值。对给定点 p，场值为 `占用` 或 `空`。

**叠加算子：** 逻辑或。

```text
V(p) = ⋁ projection_i(p)   对该关键帧所有活跃 i
```

任一点被任一投影声明即视为占用。

**Kind 参数：** 每个投影声明其占用类别：

| Kind | 含义 | 默认诊断等级 |
|---|---|---|
| `hard` | 刚性物理碰撞边界（物体的实际 AABB） | `error` |
| `soft` | 弹性风险/安全区域（摆臂半径、坠落区、粉尘/噪音包络） | `warning` |

Hard 和 soft 是**两个独立的 OccupancyField 实例**——互不混合。hard 投影注入 hard 占用场，soft 投影注入 soft 占用场。

**空间表示：** 每个投影提供几何体——AABB、球、锥、或其布尔组合（并、交、差）。编译器施加遮挡剔除：投影的有效形状为其未被结构遮挡体（楼板、墙体、柱）阻挡的部分。

**探针断言：** `V(采样范围) == 空`

- 探针定义几何采样区域（通常为活动自身的 hard 或 soft 占用包络）
- 计算值为该区域内所有活跃占用投影的并集
- 交集非空 → 断言失败
- 探针声明它采样哪种 kind 的场（hard 探针 → hard 场，soft 探针 → soft 场）

**跨 Kind 采样：** 探针可以采样与自身活动不同 kind 的场。例如：已完结构声明 hard 占用；塔吊的 soft 探针采样 hard 场，判断其风险区是否侵入结构。

**诊断等级：** 按探针声明的 kind 及其覆盖值。

**归因：** 几何体与探针采样区域重叠的所有投影。

**示例：**

- 两台塔吊的 hard 占用 AABB 重叠 → 硬碰撞 → error
- 塔吊 soft 占用（摆臂区）与相邻建筑的 hard 占用重叠 → warning
- 两个施工班组的 soft 占用（粉尘+噪音包络）重叠 → warning

### 3. 标量场（ScalarField）

**用途：** 建模空间中连续变化的标量——结构强度、养护进度、温度、沉降。

**值域：** ℝ⁺（非负实数）。

**叠加算子：** 按场实例声明。支持的算子：

| 算子 | 行为 | 适用场景 |
|---|---|---|
| `max` | V(p) = max(projection_i(p)) | 强度由最强贡献要素决定 |
| `min` | V(p) = min(projection_i(p)) | 保守估计——最差点决定 |
| `sum` | V(p) = Σ projection_i(p) | 累积贡献（叠合钢筋 + 混凝土） |
| `replace` | V(p) = projection_last(p) | 最新投影完全覆盖先前值 |

**时间行为：** 与容量场和占用场每帧重新计算不同，标量场**在关键帧之间保持持久**。某投影注入的值在投影活动结束后仍然存在，直至被显式移除或覆盖。

这建模的是物理过程：混凝土浇筑并获得强度后，即使浇筑活动已结束，强度也不会在下一个关键帧消失。

**探针断言：** `V(采样范围) ≥ 阈值`

- 探针在其声明区域内采样标量值
- 从未有投影注入过的区域 → V = 0（视为"场不存在"）
- V < 阈值 → 断言失败（包括期望场存在但 V = 0 的情况）

**诊断等级：** 默认 `error`。

**归因：** 采样区域内所有有贡献的投影，按时序和幅度排序。

**示例：**

- 基础养护：系列活动投影（绑筋 → 0，浇筑 → 初始强度，养护_t1 → 0.3×设计值，养护_t2 → 0.7×设计值，养护_t3 → 设计值）
- 上层施工活动探针采样基础区域，断言 V ≥ 设计值，养护完成后通过

### 4. 存在场（PresenceField）

**用途：** 建模带属性的标记实体的存在——许可、审批、中间产物完成、物料到场、设备/班组就位。

**值域：** 存在记录的集合。每条记录：

```text
{
  key: string,           // 场内唯一标识
  type: string,          // 类别，供探针过滤（permit | intermediate_product | material | resource | ...）
  attributes: map,       // 任意键值对
  valid_from: keyframe,  // 生效时间
  valid_until: keyframe  // 失效时间（null = 无限期）
}
```

**叠加算子：** 按 key 累积，相同 key 则覆盖。

- 记录按 key 累积
- 相同 key 的新投影完全替换已有记录
- 不同 key 的投影新增记录
- 超过 `valid_until` 的记录在关键帧求值时剔除

**空间行为：** 存在场与其他场类型一样，具有空间维度。每个投影声明自身的空间范围，记录仅在投影覆盖的空间区域内可见。全局生效的记录（如全场许可）通过声明全场空间范围的投影注入。

**统一接口原则：** 行政许可通过、物料到场、设备/班组就位等信息，全部通过相应的**活动**（行政审批活动、物流到货活动、资源调度活动）以投影方式注入 PresenceField，而非脱离活动体系单独插入记录。这为接入数字孪生、供应链系统、行政审批系统、设备/班组资源系统提供统一的活动级接口——外部数据源只需声明对应活动及其投影，即可进入 CPML 的冲突检测管线。

**探针断言：** `V(采样范围, key, filter) → bool`

- 探针声明采样范围（空间区域）、一个或多个 key、type 过滤条件、可选的属性约束
- 在采样范围内，对每个声明的 key，存在一条 type 匹配、属性匹配过滤条件、且 `valid_from ≤ 当前关键帧 ≤ valid_until` 的记录
- 所有 key 必须全部匹配——部分匹配即为失败

**诊断等级：** 默认 `error`。

**归因：** key 缺失时——应当注入该记录的活动投影。key 过期时——记录已过期的活动投影。

**示例：**

- 行政许可：行政审批活动投影 `{key: "excavation_permit", type: "permit", valid_until: KF_240}`，空间范围覆盖场地 A 区；开挖活动探针在 A 区采样，查找 `key: "excavation_permit", type: "permit"` → 存在且有效 → 通过
- 中间产物：绑筋活动投影 `{key: "wall_A_rebar", type: "intermediate_product", attributes: {status: "ready"}}`，空间范围 = wall_A 区域；浇筑活动探针在 wall_A 区域采样，查找 `key: "wall_A_rebar"`，要求 `attributes.status == "ready"` → 通过
- 物料追踪：物流到货活动投影 `{key: "steel_batch_42", type: "material", attributes: {location: "receiving_zone"}}`，空间范围 = 卸货区；搬运活动探针在卸货区采样，查找 `key: "steel_batch_42"`，要求 `attributes.location == "receiving_zone"` → 通过

## 状态持久的差异性

不同场类型在关键帧之间的状态保留方式不同：

| 场类型 | 持久行为 |
|---|---|
| CapacityField | **不持久。** 每帧仅从活跃投影重新计算。 |
| OccupancyField | **不持久。** 每帧重新计算。 |
| ScalarField | **持久。** 值在投影活动结束后继续存在。 |
| PresenceField | **持久。** 记录持续存在直至过期或被替换。 |

## 扩展

满足以下条件之一时可引入新场类型：

- 需要现有类型无法容纳的新叠加算子
- 需要新值域（如方向约束的向量场）
- 需要新的持久模型

引入新类型时，须定义全部五项：值域、叠加算子、持久行为、探针断言、诊断默认值。

为现有类型增加新算子（如为 ScalarField 增加 `multiply`）遵循相同规则——即使只有算子变化，也需完整定义各项。

## 待定

- **占用场的遮挡剔除规范：** 精确算法（raycasting 还是 shadow-volume）、遮挡体识别方式、单投影性能预算。推迟到实现阶段。
- **流水线流控的连续速率场**（饥饿/背压）。跨相邻关键帧的离散差分可能足够；若不够，后续可引入 RateField 类型。
- **投影的置信度/质量评分**——若分包商声明的容量偏乐观，场可携带置信权重。v1 范围外。
