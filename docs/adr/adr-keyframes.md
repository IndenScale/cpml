# ADR: 关键帧提取

## 背景

编译器在离散时间点（关键帧）上评估场景。需要确定如何从活动时间范围中提取关键帧。

## 决策

**基于活动边界的提取：** 关键帧从所有活动的 `timespan.start` 和 `timespan.end` 日期中提取，去重后按时间顺序排列。

**半开区间：** 活动在关键帧 K 处被视为"活跃"，当且仅当 `start <= K < end`。结束日期被排除，以避免重复计算，并精确建模活动结束的瞬间。

**日期级精度：** v1 使用 `chrono::NaiveDate`（ISO 8601 日期，如 `2026-01-10`），不包含时刻精度。后续如有需要，可扩展时间范围字段。

**为什么不用固定间隔采样？** 施工活动的状态变化发生在其边界时刻，而非任意间隔。在边界处采样能捕获所有状态转移，避免冗余计算。

## 后果

- 关键帧数量 ≤ 2 × 活动数量（去重后更少）
- 某活动结束日期与另一活动开始日期相同时，形成干净交接（边界处无重叠）
- v1 不支持日内精度；短于一天的活动暂无法建模
