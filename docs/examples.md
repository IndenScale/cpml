# CPML 场景示例

以下使用自然语言描述 CPML 各概念如何组合表达常见施工场景。

## 1. 硬硬碰撞

塔吊 A 和塔吊 B 均声明了**结构**属性，各自自动生成硬碰撞探针和硬碰撞投影。塔吊 A 的投影向碰撞场注入塔吊 A 的几何占用，塔吊 B 的硬碰撞探针对碰撞场进行采样，检测到场强异常（几何重叠）。同时也触发塔吊 B 投影被塔吊 A 探针采样的对称检测：

- 探针类型：硬碰撞探针（自动派生），采样碰撞场
- 诊断等级：`error`
- Blame：塔吊 A 的投影（注入碰撞场，被塔吊 B 的探针采样检出）或塔吊 B 的投影（被塔吊 A 的探针检出）

## 2. 软硬碰撞

塔吊活动声明了一个软碰撞投影，向软碰撞场注入吊臂摆动风险区。相邻建筑的已完成结构自动生成硬碰撞探针，采样该软碰撞场，检测到场强覆盖自身几何包络：

- 探针类型：硬碰撞探针采样软碰撞场
- 诊断等级：`warning` 或 `error`，取决于远近策略
- Blame：塔吊的软碰撞投影（在场中注入了风险覆盖）

## 3. 软软碰撞

两个施工班组分别在相邻区域作业，各自声明软碰撞投影，向软碰撞场注入各自的施工影响范围。双方均配置软碰撞探针，交叉采样对方注入的软碰撞场。探针检测到场强覆盖自身作业面，但不涉及硬碰撞：

- 探针类型：软碰撞探针采样软碰撞场
- 诊断等级：`warning`
- Blame：对方活动的软碰撞投影
- 建议：空间协调

## 4. 资源竞争

供电活动声明投影，向供电容量场注入容量（线性扣减规则）。三个施工活动各自声明显性耗电探针，对同一容量场进行采样扣减。第一和第二个施工活动探针扣减后场强仍有剩余，第三个施工活动探针采样时发现场强不足：

- 探针类型：资源消耗探针（容量场扣减）
- 诊断等级：`error`
- Blame：供电活动的投影（注入的容量不足）+ 前两个施工活动的资源消耗探针（已扣减部分容量）
- 诊断信息：期望容量 X、剩余容量 Y、缺口 Z

## 5. 行政许可

行政审批活动声明投影，向许可场注入许可信息（全局覆盖或指定空间范围，含许可类型、有效期、适用施工类型）。具体类型施工活动的探针采样许可场，检查对应类型许可是否存在且有效：

- 许可存在且有效 → 通过
- 许可不存在 → `error`，Blame 行政审批活动的投影（未注入对应许可）
- 许可已过期 → `error`，Blame 行政审批活动的投影（注入的许可有效期不足）

## 6. 工序依赖

前序活动（如墙体绑筋）声明投影，向中间产物场注入绑筋完成状态（强度 = 未达标）。后继活动（如墙体浇筑）配置探针，采样该中间产物场，检查场是否存在、场强是否达标：

- 场不存在 → `error`，Blame 前序活动的投影（未注入中间产物场）
- 场存在但强度不达标 → `error`，Blame 前序活动的投影（注入的场强不足）
- 场存在且达标 → 通过

工序依赖的前后继活动在时间轴上分离，后继必须等待前序完成。

## 7. 流水线依赖

起吊、接收、建筑内搬运三个活动在同一关键帧内并行执行，形成物料搬运流水线。起吊活动声明投影，向物料位置场注入当前起吊位置；接收活动配置探针采样物料位置场（确认物料已到达接收点位），同时声明投影注入新的物料位置；搬运活动配置探针采样接收后的物料位置场：

- 任一环节探针采样失败 → `error`，Blame 上游活动的投影（物料位置场未更新到预期位置）
- 三环节探针全部通过 → 流水线畅通

流水线依赖要求各活动在同一时间片内同步协调，而非串行交接。

**节拍与流控：** 流水线各环节可声明节拍强度（吞吐速率），探针不仅检查物料是否到位，还检查物料位置场的场强变化率：

- **饥饿（Starvation）：** 上游活动投影注入物料的速度低于下游探针期望的消耗速率，物料位置场场强持续下降 → `warning`，Blame 上游活动的投影（注入速率不足）
- **背压（Backpressure）：** 下游活动消耗速率低于上游注入速率，物料位置场场强趋近饱和上限，上游投影无法继续注入 → `warning`，Blame 下游活动的探针（消耗速率不足）

节拍匹配诊断帮助识别流水线瓶颈，指导资源配置平衡。

## 8. 通道竞争

将关键施工通道建模为容量场，初始容量场注入通行容量 N。各施工班组活动的探针不指向作业面，而是指向通道空间，采样通道容量场并扣减：

- 通道剩余容量 ≥ 1 → 通过
- 通道剩余容量 = 0 → `error`
- Blame：所有已扣减通道容量场的探针（各自扣减了部分通行容量）及其对应的投影

## 9. 基础强度

基础施工拆分为系列活动，逐步向结构场注入强度：

1. 绑筋活动 — 投影向结构场注入几何包络（强度 = 0）
2. 浇筑活动 — 投影向结构场注入初始强度
3. 养护活动 — 投影持续递增结构场强度，直至达到设计值

上层施工活动的探针采样基础结构场，检查场是否存在 + 几何是否覆盖 + 强度是否达标：

- 场不存在 → `error`，Blame 基础施工系列活动的投影（未注入结构场）
- 场存在、强度未达标 → `error`，Blame 养护活动的投影（注入的强度增量不足）
- 场存在且强度达标 → 通过
