# CPML 工作流程

## 编译与诊断管线

```mermaid
flowchart TD
  A[读取 CPML 代码文件] --> B[解析原子活动]
  B --> C[识别所有关键帧]
  C --> D{遍历所有关键帧}
  D --> E{遍历关键帧中所有原子活动}
  E --> F{遍历原子活动的所有探针}
  F --> G[计算探针采样区域的场强]
  G --> H[断言判定]
  H -->|失败| I[生成诊断<br/>分级: debug/info/warning/error/fatal<br/>归因 Blame: 追溯场贡献者]
  H -->|通过| F
  I --> J[累积指标<br/>成本/风险/进度指数]
  J --> F
  F --> E
  E --> D
  D --> K[输出诊断报告 + 指标积分曲线]
```

## 关键设计决策

**按需计算：** 场强计算仅在探针采样范围内执行（而非对全场景做全局仿真），避免不必要计算。每个探针声明其关注的空间范围和场维度，编译器只计算这些区域的场叠加结果。

**Blame 机制：** 探针告警时，编译器回溯场强的所有贡献投影，输出贡献者列表（哪个活动的哪个投影、贡献比例），支撑施工方案的根因分析。

**指标积分：** 每个关键帧的诊断按分级规则累加到指标（成本偏差、风险指数、工期影响等），输出时间序列，形成方案的可量化比较基准。
